hexo-github博客搭建

发表于 2023-01-09 分类于 愿 本文字数： 1.3k 阅读时长 ≈ 1 分钟 
小白有话说：博客来来回回建了十多次，第一次建立直接耗费了我三天的时间。

究其原因，就是中间的步骤出了点小插曲，希望大家建立博客时能够一步一个脚印，
一次性搭建完成。废话不多说，上三部曲：

一、环境准备
1.1 NodeJs安装：https://nodejs.org/zh-cn/，安装时一直点next即可。因为Hexo是基于NodeJs的，所以需要安装NodeJs。安装完成后，打开cmd输入node -v，如果出现版本号，表示安装成功。

1.2 Git安装：https://git-scm.com/downloads，安装时一路next。安装完成后，打开cmd输入git –version检查安装，如果显示的是当前git的版本号，表示安装成功。
1.3 注册Github账号：因为我们需要把Hexo站点托管到GitHub上，所以我们需要一个GitHub账号。打开GitHub官网，点击Sigh up，然后输入邮箱、密码、用户名，通过人机验证以后，点击Create account创建账号就可以了。
1.4 Github创建个人仓库：

username为你的GitHub用户名。

二、安装hexo
1、创建一个文件夹blog，在此文件夹中点击右键打开git bash here，输入命令：npm install -g hexo-cli安装hexo，安装好以后输入hexo -v 检查安装，如果输出的是一堆版本号那就安装成功了。
2、输入hexo init 命令进行hexo初始化，会发现文件里面出现了许多文件。
3、配置好了，可以输入hexo g生成静态网页，然后输入hexo s打开本地服务器来查看网页，

三、上传GitHub
我们要把我们的博客上传到GitHub，首先要连接GitHub，
首先打开Git Bash，
输入git config –global user.name “GitHub账号的名字”，
再输入git config –global user.email “GitHub账号的邮箱”
然后输入ssh-keygen -t rsa -C “GitHub账号的邮箱”来生成SSH Key，按3次回车。
在电脑文件中查找.ssh，打开pub文件复制内容，紧接着进入github：
进入settings。

Key处输入复制的公钥。
打开blog目录下的_config.yml文件，翻到最后进行修改

branch处现在一般使用main，存储库链接为GitHub的存储库链接。
然后在cmd上输入npm install hexo-deployer-git –save安装GitHub推送插件，
安装好了以后再输入hexo clean,紧接着输入hexo g生成静态网页，再输入hexo d上传到GitHub就可以了，
然后上GitHub把分支调成master/main就可以看到推送的博客文件了。
博客地址就是你的储存库名字：即username.github.io。
